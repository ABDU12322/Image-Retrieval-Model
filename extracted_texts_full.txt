A Unified Multimodal Retrieval System Combining
SimCLR and CLIP-Style Contrastive Objectives
Adeel Tahir and Abdullah Sarafraz
Department of Software Engineering
National University of Sciences and Technology (NUST), Islamabad, Pakistan
Abstract
We present a unified multimodal image retrieval system that serves both text-to-image (T2I) and image-to-image (I2I) queries
from a single shared 128-dimensional embedding space. A ResNet-50 image encoder and a DistilBERT text encoder are jointly
trained on 2,000 MS-COCO image-caption pairs using a weighted combination of SimCLR NT-Xent loss ( τ =0.10) and a CLIP-
style symmetric cross-entropy loss ( τ =0.07), with α=β=0.5. Training runs for 10 epochs on CPU, reducing total loss from 1.461
to 0.320 (78% reduction). Retrieval is served via a FAISSIndexFlatIP over L2-normalised embeddings. T2I semantic evaluation
on 50 COCO val2017 queries ( K=5, gallery = 5 ,000) achieves Average CLIP Score@5 = 0 .248, Semantic NDCG@5 = 0 .986,
and Soft Recall@5= 0.880. I2I evaluation using COCO category ground truth (50 queries, 4,902-image gallery, hard mode) yields
Category Precision@5= 0.828, Category NDCG@5= 0.838, and Category Recall@5 = 0.960, with Self-Embedding Cosine@5 =
0.780.
Index Terms
Multimodal retrieval, contrastive learning, SimCLR, CLIP, FAISS, MS-COCO, semantic evaluation
I. I NTRODUCTION
A. Problem Definition
Existing image retrieval systems are largely unimodal: they
accept either a text query or an image query, but not both,
within a single unified model. This forces practitioners to
maintain separate pipelines for text-based and image-based
retrieval, limiting flexibility and increasing operational com-
plexity in deployment.
This project builds a single multimodal retrieval model that
accepts either an image or a text query and retrieves seman-
tically similar images from the same gallery. The expected
outcome is a unified model with improved retrieval accuracy
compared to unimodal baselines, evaluated on the MS-COCO
dataset [3].
B. Research Gap
Previous work in image retrieval has focused on single-
modal approaches — using either image input only (e.g.
contrastive visual representation learning [1]) or text input only
(e.g. natural-language image search) to retrieve similar images.
No prior lightweight unified model was proposed that handles
both modalities within a single shared training objective and
a single checkpoint.
This project addresses that gap by combining image-to-
image learning (SimCLR-based contrastive learning across
augmented views) with image-to-text alignment (CLIP-style
vision-language loss) into one loss function:
L = α LSimCLR + β LCLIP, (1)
enabling one model to serve both T2I and I2I query types
effectively.
C. Contributions
We combine two complementary objectives: SimCLR [1]
builds view-invariant visual neighbourhoods; CLIP-style train-
ing [2] aligns vision with language. The joint loss trains
one encoder pair whose shared embedding space is compact
for visual similarity and semantically aligned for cross-modal
queries. Our system uses ResNet-50 + DistilBERT [5] trained
on 2,000 COCO pairs for 10 epochs on a CPU. Despite these
constraints, Soft Recall@5 = 0 .88 and Category Recall@5 =
0.96 demonstrate meaningful semantic retrieval.
II. R ELATED WORK
Unimodal image retrieval. SimCLR [1] learns view-
invariant visual representations by maximising agreement be-
tween two random augmentations of the same image via NT-
Xent loss. While it achieves strong image-to-image neighbour-
hood quality, it provides no mechanism for text-based retrieval.
Similarly, earlier contrastive methods such as MoCo and
BYOL operate purely within the image modality. These single-
modal approaches require a separate, independently-trained
text retrieval pipeline when language queries are needed,
multiplying maintenance overhead.
Unimodal text retrieval. Dense passage retrieval methods
fine-tune language models (e.g. BERT variants) to embed
queries and documents into the same space. Flickr30K [4]
and COCO Captions [3] have been widely used to benchmark
caption-to-image retrieval, but these benchmarks typically
assume a dedicated text-only query model.
Cross-modal alignment. CLIP [2] trains dual image-text
encoders with symmetric cross-entropy on 400 million web
image-text pairs, achieving remarkable zero-shot T2I retrieval.
However, CLIP is a large-scale model (ViT-B/32 has 151M
parameters) and requires substantial compute to train from

X. D ISCUSSION
SimCLR vs. CLIP contribution. SimCLR loss (final:
0.200) provides compact visual neighbourhoods that directly
benefit I2I retrieval. CLIP loss (final: 0.439) enables T2I
by aligning text and image embeddings. The joint objective
(Eq. 5) preserves both properties in one checkpoint.
Semantic gap analysis. In T2I, Soft Recall@5 = 0 .88 vs.
Exact Recall@5 = 0 .10 is a 78-point gap. This means the
model finds semantically correct content in nearly 9 of 10
queries but rarely surfaces the specific file. This is appropriate
system behaviour: in production, users prefer a semantically
matching image over an exact-match failure.
Evaluation methodology. Using frozen CLIP for T2I and
COCO category labels for I2I ensures neither evaluator is
biased by the model under test. CLIP has language-alignment
bias that inflates image-image similarity scores; COCO labels
are model-agnostic ground truth.
Limitations. Batch size 16 limits contrastive negative di-
versity. Only 2,000 training pairs are used out of 118,287
available. Exact T2I recall (0.10 at K=5) leaves significant
room for improvement with hard-negative mining or larger
training subsets.
XI. C ONCLUSION
We presented a unified multimodal retrieval system that
handles T2I and I2I queries from a single 128-D embedding
space trained with combined SimCLR and CLIP-style objec-
tives. Training on 2,000 COCO pairs for 10 epochs achieves
78% loss reduction. Semantic evaluation confirms strong per-
formance on both tasks: T2I Soft Recall@5 = 0.880, Semantic
NDCG@5= 0 .986; I2I Category Precision@5 = 0 .828, Cat-
egory Recall@5 = 0 .960. The system is fully reproducible,
CPU-feasible, and directly extensible to larger backbones,
more training data, and GPU-accelerated FAISS.

As sometimes people have description of something and they need to find image related to that, the description may be in the textual form or in image form. Taking inspiration from multi-layered and hybrid architectures commonly used in complex domains such as object classification in an image, To address this, we develop a hybrid cross-modal image retrieval system built on the principles of Contrastive Language-Image Pre-training (CLIP), which allows users to search images using either other images or natural language text queries. Instead of relying on traditional classification-based image search or manual metadata tagging, we map both visual and textual inputs into a shared 512-dimensional vector space that is same for both text and images, enabling meaningful comparisons across modalities. Our system combines two backbone architectures, one is ResNet-50 for hierarchical visual feature extraction through its deep Convolutional Neural Network (CNN) structure, and the other is Vision Transformer (ViT-B/32) for semantic understanding of text. Both connected through Adaptive Multi-Layer Perceptron (MLP) projection heads that we train using InfoNCE contrastive loss to align the result of both the models in the same vector space. For evaluation, we move beyond rigid exact-match recall and instead adopt a Foundation-Model-graded Contextual Relevance methodology, which better reflects real-world retrieval quality by judging the similarity and closeness of images in the vector space. Our results validate the effectiveness of our design achieving 100% Contextual Precision at Rank 1 (P@1ctx) and an nDCG@10 score of 0.908 on reverse image search tasks, showing that our hybrid projection architecture successfully bridges visual and textual modalities for robust multimodal image retrieval.

Introduction
We live in a world where images are everywhere. From entertainment websites to enterprise databases, the amount of visual content being generated every day has grown far more than what traditional search systems were ever designed to handle. Keyword tags break down, manual annotations don't scale, and traditional classification models simply cannot understand the way humans think about images. There must be a better way to do this.
That is exactly what this project sets out to build. We propose a hybrid, multimodal image retrieval system that lets users search a visual database in two natural ways, either by typing a plain text description or by providing an example image. Rather than forcing the system to match exact labels, tags and categories, we train it to understand  meaning of images as what is happening in the image. To do this, we combine two specialized neural network backbones: a ResNet-50 CNN that learns to extract rich visual features from images, and a Vision Transformer (ViT-B/32) that handles the semantic understanding of text. Both are brought together through jointly trained MLP projection heads into a shared 512-dimensional vector space, where similarity between any two inputs whether it is text-to-image or image-to-image is measured through cosine similarity. The entire system is trained using CLIP-based contrastive learning with InfoNCE loss, pushing semantically related pairs closer together and unrelated ones further apart.
The rest of this paper walks through everything we built and tested. Section II reviews related work, Section III lays out the mathematical foundations, Section IV details our architecture, Section V explains the contrastive learning framework, Section VI covers our experimental setup, Section VII introduces our evaluation strategy, Section VIII analyzes our results, and Section IX wraps up with conclusions and future directions. 
Literature Review
CNN-Based Visual Feature Learning
He et al. [1] introduced ResNet-50, a deep convolutional network that uses skip connections to solve gradient issues in very deep networks. The key idea is that instead of learning everything from scratch, the network learns the difference from the input: F(x) = H(x) + x. This allowed training much deeper networks successfully. ResNet-50 became widely used for extracting visual features from images. However, it was designed for classification tasks, so its features work well for categorizing images but not for measuring how similar two images are semantically—which is what we need for retrieval.
Contrastive Learning for Similarity
   Chen et al. [2] introduced SimCLR, a framework that teaches models to understand similarity rather than just classification. It uses NT-Xent loss to pull similar samples closer together in feature space while pushing dissimilar ones apart. He et al. [3] improved this with MoCo, which made contrastive learning more efficient by using a momentum encoder and memory bank, avoiding the need for very large batch sizes. These methods produced more flexible representations than classification-based training, but they only worked within one type of data—images were only compared to other images, never to text descriptions.
Vision Transformers for Semantic Understanding
  Dosovitskiy et al. [4] proposed the Vision Transformer (ViT), which applied the Transformer architecture—originally designed for language by Vaswani et al. [5]—to images. Instead of using convolutional filters, ViT splits images into patches and processes them as a sequence, capturing long-range relationships across the entire image. Research has shown that combining CNNs with Transformers works better than using either alone [6], because CNNs are good at local details while Transformers excel at understanding global context. Our work takes advantage of this by using both ResNet and ViT together.
Multimodal Learning
Radford et al. [7] introduced CLIP, which extended contrastive learning to connect two different types of data—images and text. CLIP was trained on millions of image-text pairs from the internet, using InfoNCE loss [8] to align images and text in a shared space. This alignment is measured using cosine similarity: sim(v,t) = (v·t)/(||v|| ||t||). This breakthrough showed that we could bridge vision and language, enabling text-based image search.

Related Work in Multimodal Image Retrieval
   Recent work has explored different aspects of multimodal retrieval, but most approaches have significant limitations. Moro et al. [9] and Zhou and Hudin [10] developed domain-specific solutions for fashion retrieval, achieving good results but lacking generalization to other domains. Naik and Agarwal [11] improved CLIP's robustness under challenging conditions through their ENCLIP framework, while Bibi and Rehman [12] confirmed that feature space quality directly impacts retrieval performance. However, existing methods either focus on single domains, use single-modal backbones, or treat text-to-image and image-to-image retrieval as separate problems. Our work addresses these gaps by combining ResNet-50 and ViT-B/32 through jointly trained projection heads, creating a unified system that handles both retrieval modes in one architecture.
Proposed Hybrid Architecture
To process fundamentally different data modalities effec-tively, we design a dual-encoder architecture featuring robust, state-of-the-art foundation models acting as feature extractors. These backbones are coupled with actively trained Multi-Layer Perceptron (MLP) projection heads. This hybrid multi-layered approach ensures that the model benefits from immense pre-trained world knowledge while still retaining the flexibility to adapt to the geometry of our specific dataset.

Contrastive Learning Framework
The defining characteristic of our architecture is the loss function used to optimize the projection heads. We utilize the InfoNCE (Information Noise-Contrastive Estimation) loss.
A. InfoNCE Loss Formulation
During training, the system processes data in batches of size
N , yielding N image-text pairs. The forward pass generates N image embeddings (I1, I2, . . . , IN ) and N text embeddings (T1, T2, . . . , TN ).
The model computes an N × N cosine similarity matrix. The diagonal elements (Ii ·Ti) represent the ”positive” ground-truth pairs, while all off-diagonal elements (Ii · Tj for i ̸= j) represent ”negative” pairs.
The loss function attempts to maximize the similarity of the positive pairs while minimizing the similarity of the N − 1 negative pairs. This is achieved by computing a symmetric categorical cross-entropy loss over the similarity matrix.

Evaluation Methodology
Evaluating a multi-modal retrieval system requires nuance. This section outlines the shift from traditional rigid metrics to semantic, contextual evaluations.
The Limitations of Exact Match Recall
Standard retrieval metrics, such as Recall@K (R@K), strictly require the system to retrieve the exact paired image provided in the ground-truth dataset. In highly diverse datasets, this introduces a severe false-negative penalty. If a user queries ”A brown dog playing in the park,” and the model retrieves a perfect image of a brown dog in a park that happens to be Image B, but the dataset paired the query with Image A, the Recall metric scores a 0. This rigid evaluation fails to measure the true semantic capability of the model.
Foundation-Graded Contextual Relevance
To address this flaw, we developed a contextual evaluation pipeline that utilizes an external, frozen foundation model as a ”judge”.
Foundation Grading: The retrieved results and the query are passed through a massive, perfectly converged Foundation CLIP model to compute an absolute seman-tic similarity score (CLIPScore).
Contextual Precision (P@Kctx): We establish a strict semantic threshold of 0.60. Any retrieved item that scores above 60% semantic similarity is marked as a ”correct” contextual match, regardless of whether it is the exact ground-truth pair.
Normalized Discounted Cumulative Gain (nDCG@K): We compute the ranking quality based on these graded semantic scores, penalizing the model if highly relevant images are ranked below moderately relevant ones.
Results and Discussion
The model was evaluated across two distinct retrieval paradigms: Image-to-Image (visual similarity) and Text-to-Image (semantic similarity). The results highlight a stark contrast between intra-modal clustering and cross-modal align-ment under constrained compute.
Image-to-Image Retrieval Analysis
For Reverse Image Searching (Image-to-Image), the text encoder is bypassed entirely. A query image is passed through the ResNet-50 Image Encoder, and FAISS is used to retrieve the nearest neighbor image embeddings.
The architecture proved extraordinarily effective at mapping similar visual concepts. As detailed in Table I, the model achieved a Contextual Precision at Rank 1 (P@1ctx) of 1.000. This indicates that 100% of the top retrieved images success-fully exceeded the strict 60% semantic similarity threshold. The average semantic similarity of the top results (CLIP-Score@1) was 0.743. Furthermore, the nDCG@10 of 0.908 demonstrates near-perfect ranking geometry within the visual latent space, proving that the multi-layer MLP projection head successfully learned to organize visual data logically.

Limitations
The disparity between Image-to-Image and Text-to-Image performance is directly attributable to the batch size bottle-neck. Contrastive learning relies heavily on in-batch negative sampling. With a batch size of only 16, the model only sees

15 negative examples per step. This prevents the loss function from creating sufficient repulsive force to perfectly align the complex text embeddings with the image embeddings, resulting in the ”under-baked” 57% similarity ceiling.
Conclusion and Future Work
This paper presented a highly structured, hybrid multi-layered approach to cross-modal semantic image retrieval. By integrating a deep CNN visual backbone with a Transformer text backbone and mapping them through adaptive projection heads via contrastive learning, we constructed a system capa-ble of robust semantic search.
We demonstrated that despite restricted computational re-sources, the visual latent space organizes beautifully, achiev-ing 100% contextual precision in reverse image searching paradigms. We also introduced a more accurate, foundation-graded evaluation methodology that solves the flaws of rigid exact-match metrics.
To close the semantic gap in text-to-image performance and push the cross-modal precision above 90%, future work will focus on three key architectural and training optimizations:
Parameter-Efficient Fine-Tuning (PEFT): Implement-ing Low-Rank Adaptation (LoRA) within the ViT and ResNet layers to accelerate the convergence of the deep backbone features without catastrophic memory over-head.
Hard Negative Mining: Modifying the FAISS vector database to dynamically feed highly similar but seman-tically distinct images into the training batches. This will aggressively penalize the model for confusing subtle visual differences, forcing the projection heads to learn finer discriminative boundaries.
Compute Scaling and Batch Expansion: Utilizing dis-tributed cloud GPU infrastructure to scale the batch size from 16 to 1024 or higher, providing the InfoNCE loss with sufficient negative examples to properly orient the text-to-image embedding space over 100+ epochs.
